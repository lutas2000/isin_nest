# Skill：權限變更與授權保護

## 目的

降低權限調整造成的越權風險，確保授權行為可驗證。

## 觸發條件

- 任務包含角色權限調整、Guard 變更、敏感 API 存取控制

## 依賴規則

- `../rules/backend/security-authz.md`
- `../rules/backend/api-contract-swagger.md`

## 輸入

- 權限變更目標（誰可做什麼）
- 影響 endpoint 清單
- 既有角色與功能矩陣

## 執行步驟

1. 定義最小權限與影響範圍。
2. 在 endpoint 層加上 Guard/授權檢查。
3. 補齊 401/403 行為文件與回應語意。
4. 驗證未登入、無權限、有權限三類情境。
5. 確認既有角色未被意外放大權限。

## 輸出

- 更新後授權邏輯
- 權限驗證結果
- 契約文件同步結果

## 完成後文件更新檢查

- 若權限模型新增新概念：更新 `security-authz.md`
- 若驗證流程有新共識：更新本 skill
