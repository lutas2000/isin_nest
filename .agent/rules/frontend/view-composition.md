# 前端規則：View 組裝與頁面結構

## 適用情境

- 新增或重構 `views/*` 頁面
- 將大型頁面拆分為可維護結構

## 必做步驟

1. 先定義頁面責任與子元件邊界。
2. 將 UI 區塊拆為可重用元件，頁面只保留流程協調。
3. 狀態來源（local state / store / API）明確分層。
4. 路由頁需確認是否設置 `meta.requiresAuth`。

## 禁止事項

- 不可在單一頁面堆疊過多商業邏輯與樣式細節。
- 不可繞過既有 API service 層直接在頁面呼叫 `fetch`。

## 驗證方式

- 頁面可讀性提升，元件責任清楚。
- 路由權限與登入流程行為正確。
- 主要互動無回歸問題。
