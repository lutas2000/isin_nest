# 前端規則：API 與狀態管理

## 適用情境

- 新增前端 API 呼叫
- 調整 Pinia 狀態流
- 錯誤處理與登入狀態流程調整

## 必做步驟

1. API 呼叫統一走 `services/api.ts` 封裝。
2. 將跨頁共用狀態放入 store，頁面狀態保留在元件內。
3. 對 401/403/業務錯誤定義一致處理策略。
4. 變更 API 契約時，同步檢查對應 service 與 UI 顯示邏輯。

## 禁止事項

- 不可在 view 或 component 直接散落重複 API 呼叫邏輯。
- 不可在多處各自實作不一致的錯誤訊息策略。

## 驗證方式

- API 呼叫點可追蹤且集中。
- 錯誤顯示與登入失效流程一致。
- 相關頁面與 store 行為符合預期。
