# 品質檢核門（Quality Gates）

本文件將「任務完成後思考是否更新文件」落地為可執行檢核流程。

## Gate 1：任務前檢核

- 是否已讀取 `Agent.md` 與 `rules-router.md`
- 是否明確辨識任務類型與必讀規則
- 是否只加載必要文件（避免全量讀取）

## Gate 2：任務中檢核

- 是否遵守所讀規則中的禁止事項
- 是否在關鍵變更點同步更新文件草稿（避免收尾遺漏）
- 是否有跨領域影響，需補讀其他分類規則

## Gate 3：任務後檢核

- 是否完成功能驗證（最小可行驗證即可）
- 是否依 `doc-update-decision-tree.md` 判斷文件更新需求
- 是否更新了受影響的規則與技能文件

## 最小交付要求

- 回報本次任務讀取了哪些規則文件
- 回報是否更新文件，若未更新需說明理由
- 若新增可重用流程，應建立或更新 `.agent/skills/*`
