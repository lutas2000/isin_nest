# Docker 規則：本地開發 Compose

## 適用情境

- 調整本地開發用 `docker-compose.yml`
- 建立或修改本地服務依賴（DB、快取、後台工具）

## 必做步驟

1. 確認本地開發必要服務清單與啟動順序。
2. 以最小配置啟動，避免不必要的重量級服務。
3. 為關鍵服務設定健康檢查或可觀察啟動訊號。
4. 調整後驗證開發流程可在新成員環境重現。

## 禁止事項

- 不可將機器專屬路徑硬編碼進 compose 設定。
- 不可讓開發 compose 與專案啟動流程嚴重脫鉤。

## 驗證方式

- 本地一鍵啟動可用，且服務間連線成功。
- 文件與實際 compose 行為一致。
