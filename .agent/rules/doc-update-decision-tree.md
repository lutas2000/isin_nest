# 文件更新判斷樹

用於任務完成後快速判定「是否必須更新指導文件」。

## 判斷流程

1. 本次任務是否新增了新的作業模式？
   - 是 -> 必須更新 `.agent/rules/*` 或新增 `.agent/skills/*`
   - 否 -> 進入下一題
2. 既有規則是否與實作行為不一致？
   - 是 -> 必須修正對應規則文件
   - 否 -> 進入下一題
3. 本次任務是否暴露高頻錯誤或反覆手動步驟？
   - 是 -> 應新增或更新 skill 文件
   - 否 -> 進入下一題
4. 是否有路由映射遺漏（任務找不到對應規則）？
   - 是 -> 必須更新 `rules-router.md`
   - 否 -> 可不更新，並於任務回報中記錄「本次不需更新」

## 必須更新的情境

- 任務類型首次出現
- 規則與現況衝突
- 技能可顯著降低重工
- 跨領域任務但路由未覆蓋

## 可延後更新的情境

- 純小修、且規則與技能均完整涵蓋
- 暫時性除錯，不構成可重用流程

## 更新優先順序

1. `rules-router.md`（先確保可被找到）
2. 對應分類規則（確保可被執行）
3. `.agent/skills/*`（沉澱可重用流程）
4. `Agent.md`（僅更新入口索引）
